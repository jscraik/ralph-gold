# Project Structure & Scaffolding

## Purpose

Define the Ralph Gold repository layout and the scaffolding generated by `ralph init`.

## Repository Layout (source-of-truth)

```
.
├─ src/ralph_gold/        # Core library + CLI implementation
├─ tests/                # Pytest suite
├─ docs/                 # User and contributor documentation
├─ scripts/              # Helper scripts
├─ vscode/ralph-bridge/   # VS Code bridge extension
├─ .ralph/               # Loop config/state for this repo (local)
└─ ...
```

### Core Code

- `src/ralph_gold/`: CLI, orchestration logic, and supporting modules.
- `src/ralph_gold/templates/`: scaffolding templates used by `ralph init`.
- `src/ralph_gold/scaffold.py`: orchestration for generating the scaffolded files.

### Documentation

- `docs/`: product docs, usage guides, and protocol references.
- `docs/assets/`: images and other documentation assets.

## Scaffolding (ralph init)

`ralph init` creates a `.ralph/` directory with default files that drive the loop:

- `.ralph/ralph.toml` (config)
- `.ralph/PRD.md` or `.ralph/prd.json` (task tracker)
- `.ralph/AGENTS.md` (repo-specific build/test/run commands)
- `.ralph/progress.md` (append-only progress log)
- `.ralph/specs/` (requirements specs)
- `.ralph/PROMPT_*.md` (agent prompts)
- `.ralph/logs/`, `.ralph/receipts/`, `.ralph/context/` (run artifacts)

The scaffold templates are the single source of truth for these files and are stored in
`src/ralph_gold/templates/`.

## Smoke Test

Run the CLI help to confirm the entrypoint is wired:

```bash
uv run python -m ralph_gold.cli --help
```

## Where To Add New Work

- Core features: `src/ralph_gold/`
- Tests: `tests/`
- Docs: `docs/`
- VS Code bridge changes: `vscode/ralph-bridge/`
